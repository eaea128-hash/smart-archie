# Skill: Vendor Evaluation（廠商評估）

**觸發時機**：「比較廠商」、「選雲端」、「RFP」、「RFI」、「採購」、「哪個比較好」

## RFI vs RFP 差異

| 文件 | 目的 | 時機 |
|------|------|------|
| **RFI** (Request for Information) | 市場探索、了解廠商能力 | 需求還不確定、前期調研 |
| **RFP** (Request for Proposal) | 正式徵求報價與解決方案 | 需求明確、準備採購 |

## RFI 必要結構
```
1. 公司背景與現況（2頁）
2. 資訊需求清單（廠商能力問卷）
3. 回覆截止日期與聯絡窗口
4. 說明：此為探索性文件，不保證採購
```

## RFP 必要結構
```
1. 執行摘要與採購目的
2. 現況說明（AS-IS）
3. 需求規格（功能性 + 非功能性）
4. 評分標準（權重表）
5. 報價格式要求
6. 時程與里程碑
7. 廠商資格要求
8. 合約條款要點
```

## 雲端廠商比較矩陣

| 評估維度 | 權重 | AWS | Azure | GCP | 說明 |
|---------|------|-----|-------|-----|------|
| 服務廣度 | 20% |  |  |  | IaaS/PaaS/SaaS 覆蓋率 |

## 執行步驟

1. **釐清需求**：工作負載類型、合規要求、預算上限
2. **發 RFI**：收集 3-5 家廠商初步資訊（2週）
3. **短名單**：篩選 2-3 家進行 PoC
4. **PoC 評估**：實際跑特定 workload，記錄效能與費用
5. **發 RFP**：正式報價（僅短名單廠商）
6. **評分**：依矩陣量化，加入質化說明
7. **決策備忘錄**：記錄選擇理由（未來稽核用）

