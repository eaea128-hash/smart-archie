# Skill: Change Management Plan（變革管理）

**觸發時機**：「推不動」、「組織阻力」、「變革計畫」、「上線前準備」、「導入策略」

## 核心認知
技術方案再完美，人不動等於零。
雲端遷移失敗 70% 不是技術問題，是人與流程問題。

## Kotter 8 步驟框架

```
1. 建立緊迫感    → 數據說話：現況成本、競爭威脅、技術債
2. 組建領導聯盟  → 找出 sponsor、champion、early adopter
3. 建立願景      → 一句話說清楚：「遷移後，我們能做到 X」
4. 溝通願景      → 重複 × 7 個管道（Email/會議/Slack/海報）
5. 授權行動      → 移除障礙（流程、權限、預算審批）
6. 創造短期勝利  → 先找最容易的系統做 MVP，快速展示成果
7. 鞏固成果      → 用數據強化信心，擴大範圍
8. 深植文化      → 把新做法寫進標準流程、KPI、招募條件
```

## 執行步驟

1. **評估阻力**：問「誰會反對？為什麼？他們的核心顧慮是？」
2. **繪製利害關係人地圖**：
   - 支持者（盟友）→ 讓他們發聲
   - 觀望者 → 用數據和 pilot 成果轉化
   - 反對者 → 理解痛點，個別溝通
3. **制定溝通計畫**：各角色聽什麼、看什麼、關心什麼
4. **規劃 Quick Win**：6 週內展示具體成果

## 輸出格式
```
📣 變革計畫：[專案名稱]
緊迫感訊息：[數據 + 故事]
關鍵 Sponsor：[誰]
最大阻力：[誰 / 什麼顧慮]
Quick Win 選擇：[第一個要做的系統/流程]
溝通時程：
  Week 1-2: [動作]
  Week 3-4: [動作]
  Week 5-6: [展示成果]
```
