# Skill: Debug First

**觸發時機**：使用者回報 bug、錯誤、「沒有效果」、「顯示不對」

## 執行步驟

1. **重現問題**：先確認問題的完整描述（什麼頁面、什麼操作、什麼現象）
2. **縮小範圍**：
   - 前端問題 → 先看 browser console 錯誤
   - 後端問題 → 先看 Netlify function logs
   - 資料問題 → 先確認 Supabase 回傳值
3. **讀取相關檔案**：只讀取與問題直接相關的區段，不全文掃描
4. **提出假設**：列出 2-3 個可能原因，標注最可能的
5. **最小修復**：優先用最小改動解決問題，不順手重構無關程式碼
6. **驗證**：修改後說明如何確認問題已解決

## 禁止行為
- 不在未確認根因前就開始修改
- 不同時修改超過 2 個不相關的地方
- 不猜測性修改（改了不知道為什麼）
