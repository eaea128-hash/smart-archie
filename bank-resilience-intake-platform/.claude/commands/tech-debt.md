# Skill: Technical Debt Analysis

**觸發時機**：`/tech-debt`、技術債、程式碼品質、重構、維護困難

## 四大負債類型

### 1. 程式碼負債
- 重複邏輯（DRY 違反）
- 函數過長（> 50 行）
- Magic number / hardcoded 值
- 命名不清楚的變數或函數

### 2. 架構負債
- 前後端職責不清
- API 設計不一致
- 缺少錯誤邊界

### 3. 測試負債
- 無測試覆蓋的核心邏輯
- 只有人工測試紀錄

### 4. 文件負債
- CLAUDE.md / BUGS.md 過時
- 函數無說明，邏輯難以理解

## 執行步驟

1. 掃描指定檔案或模組
2. 依四大類型列出問題清單
3. 評估優先級（Critical / High / Medium / Low）
4. 估算修復成本（小時）
5. 提出 Quick Win（1-2 天可完成）清單

## 輸出格式

```
## 技術債報告

### Critical（立即處理）
- [問題描述] → 影響：[說明] → 修復成本：[X 小時]

### Quick Wins（本週可做）
- [問題描述] → 修法：[建議]
```

## 規則
- 只分析，不自動修改
- 修復前必須執行 `/before-change`
- 優先處理影響功能正確性的負債
